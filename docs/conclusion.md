# Conclusion

This project set out to build a full-featured online vehicle showroom and marketplace platform, and the final system successfully delivers on that goal. Built with Next.js (App Router) on the frontend/backend and Supabase (Postgres, Auth, Storage, and Realtime) as the data and identity layer, the platform brings together vehicle browsing and comparison, a peer-to-peer used-vehicle marketplace, rental request handling, AI-assisted recommendations via the Claude API, and a dedicated admin panel — all under a single, coherent architecture.

Over the course of development, the project covered the full lifecycle of a real-world web application: schema design with row-level security, dual authentication systems (Supabase Auth for regular users and a custom signed-cookie session for admins), file storage for listing photos, database triggers for real-time notifications, and a search history feature with deduplicated query tracking. Each of these subsystems was verified through the unit test cases documented in this report, covering signup/login flows, listing creation and access control, test-drive booking rules, notification delivery, admin authorization, and rental request handling — all of which passed as expected.

Several design decisions proved central to the platform's reliability. Separating user and admin authentication kept privilege boundaries clear and auditable, while RLS policies ensured that users could only modify their own data without relying on application-level checks alone. Database triggers for test-drive bookings and notifications kept business rules (such as blocking bookings on sold listings) enforced consistently at the data layer rather than scattered across UI code.

The project also served as a practical exercise in working with a modern full-stack toolchain: integrating a managed backend-as-a-service (Supabase) with a React-based frontend, handling authentication edge cases such as free-tier project pauses, and structuring a growing codebase (dashboard, history, rentals, used listings, admin CRUD) without losing maintainability.

Looking ahead, the platform has clear room to grow — payment integration for rentals and purchases, more advanced search and filtering, richer analytics on the admin dashboard, and automated end-to-end testing would all strengthen it further. As it stands, however, the system meets its core objective: providing a functional, secure, and extensible platform for browsing, comparing, and transacting on vehicles, backed by a tested and well-documented codebase.
